PHASE 2 – Build the Production Management System

Use the migration analysis completed in Phase 1 as the implementation guide.

Do not redesign the website or remove any existing functionality. Preserve the current UI, user experience, products, authentication, payment flow, AI tools, subscriptions, downloads, admin dashboard, SEO tools, and all existing features.

This phase is focused on transforming the application into a production-ready platform that can be managed almost entirely from the Super Admin Dashboard without requiring code edits or direct server access. The application should also be portable to Hostinger Business Hosting or a Hostinger VPS.

---

1. Build a Secure System Configuration Centre

Create a new System Configuration section inside the Super Admin Dashboard.

This will become the central control panel for all application configuration.

Allow secure management of:

- Paystack Secret Key
- Paystack Public Key
- Clerk Secret Key
- Clerk Publishable Key
- OpenAI API Key
- Gemini API Key
- Resend API Key
- Session Secret
- JWT Secret
- Encryption Key
- Any future API credentials

Requirements:

- Encrypt secrets at rest.
- Never store secrets in plain text.
- Only Super Admin can access.
- Show/Hide values.
- Edit and Save.
- Validate before saving.
- Test Connection button.
- Display connection status.
- Display last updated date.
- Display who updated the configuration.
- Keep an audit log of every configuration change.

Build a Complete Coupon Management System

Implement a full coupon and discount system.

The Super Admin should be able to:

- Create unlimited coupons.
- Edit coupons.
- Delete coupons.
- Activate or deactivate coupons.
- Generate random coupon codes.
- Create custom coupon codes.
- Set percentage discounts.
- Set fixed-value discounts.
- Apply coupons to all products or selected products.
- Set minimum purchase amount.
- Set maximum discount amount.
- Set usage limits.
- Limit usage per customer.
- Set start and expiry dates.
- Restrict coupons to logged-in users if desired.
- View coupon analytics.
- View redemption history.
- Enable or disable coupons instantly.

Customers should be able to apply coupon codes during checkout.

Build a Complete Referral System

Create a production-ready referral programme.

Every registered user should automatically receive:

- A unique referral code.
- A referral link.
- A referral dashboard.

Allow the Super Admin to configure:

- Reward type.
- Reward amount.
- Percentage reward.
- Fixed reward.
- Store credit.
- Free product rewards.
- Minimum purchase requirement.
- Campaign start date.
- Campaign end date.
- Maximum referral rewards.
- Enable/Disable referral programme.

The system must automatically:

- Track referrals.
- Track pending referrals.
- Track completed referrals.
- Track rewards.
- Prevent duplicate referrals.
- Prevent fraud.
- Record referral history.
- Notify users of rewards.

Create an Admin Referral Dashboard showing:

- Total referrals.
- Pending referrals.
- Successful referrals.
- Conversion rate.
- Top referrers.
- Reward history.

The system must integrate seamlessly with payments, products, subscriptions, and user accounts.

---

10. Security Centre

Implement all security recommendations discovered during Phase 1.

Specifically:

- Remove hardcoded secrets.
- Remove insecure SESSION_SECRET fallback behaviour.
- Encrypt sensitive credentials.
- Eliminate plaintext credentials.
- Validate secrets during application startup.
- Prevent the application from running with insecure defaults.
- Add configuration validation checks.
- Log all security-related configuration changes.

- Database Status

- AI Services Status
- Email Service Status
- Authentication Status
- Storage Status
- Server Status

Each service should clearly display:

- Healthy
- Warning
- Error

12. Emergency Recovery Centre

Create one-click recovery tools.

Include:

- Verify All Services
- Reload Configuration
- Repair Configuration
- Refresh API Connections
- Verify Payment Gateway
- Verify Authentication
- Verify AI Providers
- Verify Email Service

These tools should safely recover configuration issues without affecting products, users, purchases, subscriptions, or other business data.

---

13. Hostinger Portability

Refactor the project so it no longer depends on Replit-specific infrastructure.

Prepare the application to run cleanly on Hostinger Business Hosting or a Hostinger VPS.

Configuration, uploads, storage, API integrations, and system settings should remain portable across hosting providers.

---

14. Validation and Testing

When Phase 2 is complete:

- Test every existing feature.
- Test payment processing.
- Test authentication.
- Test AI integrations.
- Test email delivery.
- Test downloads.
- Test products.
- Test subscriptions.
- Test the new Coupon System.
- Test the new Referral System.
- Verify that no existing functionality has been broken.

Finally, generate a comprehensive implementation report containing:

- Files created.
- Files modified.
- Security improvements implemented.
- New database changes.
- New Admin Dashboard pages.
- New API endpoints.
- Remaining recommendations before Phase 3.

This phase must not modify, migrate, reset, overwrite, or delete customer records, products, purchases, subscriptions, payment history, download history, analytics, or any other business data. Phase 3 will focus exclusively on protecting and preserving those assets across deployments, Git operations, backups, restores, and future migrations.
