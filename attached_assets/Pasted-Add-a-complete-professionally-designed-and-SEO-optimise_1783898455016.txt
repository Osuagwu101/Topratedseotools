## 4. Automatic image optimisation

Build a reliable automatic image-optimisation workflow for every blog image uploaded through the CMS.

When an administrator uploads a featured image or an image inside an article:

1. Validate that it is an approved image type.
2. Reject unsafe or corrupted files.
3. Display the maximum permitted upload size.
4. Automatically correct image orientation.
5. Remove unnecessary metadata, including EXIF metadata, where appropriate.
6. Resize oversized images while preserving their aspect ratio.
7. Compress the image without noticeably damaging visual quality.
8. Convert the image to WebP, or AVIF where fully supported, while keeping a suitable fallback.
9. Generate responsive image variants for small, medium and large screens.
10. Generate an optimised thumbnail for blog cards and admin previews.
11. Store the optimised version rather than serving the original large image.
12. Prevent duplicate unnecessary image files.
13. Show upload and optimisation progress.
14. Show a clear error message if optimisation fails.
15. Allow administrators to replace or remove an image.

Use practical target dimensions such as:

* Featured article image: maximum width of approximately 1600 pixels.
* In-article image: maximum width of approximately 1400 pixels.
* Blog-card thumbnail: approximately 600–800 pixels wide.
* Author image: approximately 300 pixels square.

Do not upscale small images.

Use responsive `srcset` and `sizes` attributes so mobile users do not download desktop-sized images.

Add explicit width and height values or an aspect-ratio container to prevent layout movement while images load.

Use lazy loading for images below the fold. Do not lazy-load the main featured image when it is the Largest Contentful Paint element. Preload or prioritise that image appropriately.

If Base44 cannot perform all transformations natively, implement the best secure alternative using a supported server-side function or an image service such as Cloudinary or ImageKit. Keep any API secret on the server and never expose it in frontend code.

Include a CMS setting for:

* Preferred output format.
* Compression quality.
* Maximum image dimensions.
* Automatic filename cleaning.
* Automatic alt-text suggestion.

The final image workflow must reduce file size, maintain good visual quality and prevent large uploads from slowing down the website.
